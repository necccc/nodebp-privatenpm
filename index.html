Per-project configuration

package.json
use ‘publishConfig’

you can set a project to use a private reg several ways

Publish - unpublish



								npm publish --registry http://your-own.registry.url modulename

							


								npm unpublish --force --registry http://your-own.registry.url modulename
its as easy as you would do it with the regular npm, just publish,
if the module you're publishing is bound to the private registry (npmrc or publishconfig), it will be available only there

unpublishing is also done the same way npm does, and if you use the --force parameter, you can purge cached public modules from the sinopia mirror

Security

Ours only accessible
from a well defined domain / ip pool

the only security setting sinopia itself has, was mentioned before, when i was talking about authenticaton:
you can tell which users can publish modules to the sinopia registry
but apart from this, anyone can access the server, even your private modules published on it
so for security reasons, we allow access to it only from our company ip pool, so we, and the deployment servers can use it
